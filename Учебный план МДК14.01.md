### 1. ПОЯСНИТЕЛЬНАЯ ЗАПИСКА

1.1. Место МДК в программе
МДК.14.01 является ключевым завершающим профессиональным модулем, интегрирующим компетенции по разработке программного обеспечения. Он формирует у студентов системный подход к обеспечению качества ПО через практические навыки тестирования и верификации, что критически важно для успешного прохождения преддипломной практики и выполнения ВКР.

1.2. Цель освоения МДК
Формирование у студентов профессиональных компетенций по планированию, проектированию, выполнению и документированию процессов тестирования программного обеспечения на всех уровнях с использованием современных инструментов, фреймворков и методологий.

1.3. Технологический стек и инструментальная база (сквозное использование):
*   IDE: Visual Studio Code с необходимыми расширениями (Python, Pylance, GitLens, Test Explorer, Markdown All in One).
*   Язык программирования: Python (для автоматизации тестов, работы с API, утилит).
*   Система контроля версий: Git, хостинг — GitHub.com.
*   Формат документирования: Markdown (.md файлы), отчеты в формате .pdf.
*   Процесс работы: Весь практикум строится на принципах CI/CD: код и тесты хранятся в Git, документация — в репозитории, изменения вносятся через ветки (feature-branch) и Pull Requests.

1.4. Требования к результатам освоения МДК (на основе ФГОС СПО)
В результате освоения МДК студент должен уметь:
*   Анализировать требования и проектировать тестовые сценарии.
*   Выполнять все виды ручного функционального тестирования.
*   Разрабатывать и исполнять автоматизированные тесты (API, UI).
*   Работать с системами управления тестированием (TestRail, Qase) и баг-трекинга (Jira).
*   Оформлять профессиональную тестовую документацию на всех этапах.
*   Использовать Git и GitHub для командной работы над тестами.
*   Интегрировать тесты в процесс CI/CD (на примере GitHub Actions).

1.5. Формы контроля
*   Текущий контроль: Защита лабораторных работ, проверка Pull Requests в GitHub, тесты по теории.
*   Промежуточная аттестация: Защита этапов курсового проекта.
*   Итоговый контроль: Комплексный экзамен в форме публичной защиты итогового проекта по тестированию с демонстрацией полного цикла работ: тест-план, набор тестовой документации, баг-репорты, код автотестов, отчет.

### 2. ТЕМАТИЧЕСКИЙ ПЛАН И СОДЕРЖАНИЕ УЧЕБНОЙ ДИСЦИПЛИНЫ

#### РАЗДЕЛ 0. ТЕХНОЛОГИЧЕСКИЙ ИНСТРУМЕНТАРИЙ И ПРОЦЕССЫ (8 часов)
*   Тема 0.1. Введение в профессию. Современный стек QA-инженера. Agile/Scrum, роль тестировщика в команде. (2 ч)
    *   *Лекция-дискуссия. Практика: регистрация на GitHub, создание репозитория для портфолио.*
*   Тема 0.2. Настройка профессионального окружения: VS Code, Python, виртуальное окружение, пакетный менеджер pip. (2 ч)
    *   *Лабораторная работа: установка и конфигурация, написание первого скрипта.*
*   Тема 0.3. Рабочий процесс на Git/GitHub: clone, commit, push, pull, ветвление (GitFlow-lite), создание Pull Request (PR). (2 ч)
    *   *Практикум: работа в парах над общим репозиторием, создание и мерж PR.*
*   Тема 0.4. Техническое документирование в разработке. Язык разметки Markdown. Структура README.md. (2 ч)
    *   *Лабораторная работа: создание профессионального README для проекта, ведение конспектов в .md.*

#### РАЗДЕЛ 1. ОСНОВЫ РУЧНОГО ФУНКЦИОНАЛЬНОГО ТЕСТИРОВАНИЯ (22 часа)
*   Тема 1.1. Жизненный цикл разработки ПО (SDLC) и место тестирования (STLC). Классификация видов тестирования. (2 ч)
*   Тема 1.2. Анализ требований (User Story, Use Case).
	Техники тест-дизайна: классы эквивалентности, граничные значения, предугадывание ошибок. (4 ч)*  
*	Практикум: применение техник к функциональным требованиям из реального ТЗ.*
*   Тема 1.3. Тест-кейсы и чек-листы. Атрибуты хорошего тест-кейса. Приоритезация. Инструменты: TestRail / Qase.io. (4 ч)
    *   *Лабораторная работа: создание набора тест-кейсов в выбранной системе.*
*   Тема 1.4. Дефекты. Жизненный цикл бага. Написание эффективного баг-репорта. Инструменты: Jira, GitHub Issues. (4 ч)
    *   *Ролевая игра "Триада": студенты по очереди выступают как тестировщик, разработчик, менеджер.*
*   Тема 1.5. Тестирование документации, юзабилити, совместимости (кросс-браузерное). (4 ч)
    *   *Практикум: тестирование веб-сайта колледжа на разных браузерах и мобильных разрешениях.*
*   Тема 1.6. Тест-план и отчет о тестировании. Метрики (Test Coverage, Defect Density). (4 ч)
    *   *Лабораторная работа: создание шаблона тест-плана и отчета в Markdown/Google Docs.*

#### РАЗДЕЛ 2. АВТОМАТИЗАЦИЯ ТЕСТИРОВАНИЯ НА PYTHON (30 часов)
*   Тема 2.1. Введение в автоматизацию. Пирамида тестирования. Фреймворк pytest: установка, структура, фикстуры (@pytest.fixture). (6 ч)
    *   *Лабораторная работа: написание юнит-тестов для функций-калькуляторов.*
*   Тема 2.2. Паттерн Page Object Model (POM) для UI-тестов. Инструменты: Selenium WebDriver, pytest. (8 ч)
    *   *Практикум: автоматизация критического пути (логин-действие-выход) на учебном сайте (e.g., saucedemo.com).*
*   Тема 2.3. Тестирование REST API. Инструменты: requests, pytest. Форматы JSON, статус-коды. (8 ч)
    *   *Лабораторная работа: создание коллекции автотестов для публичного API (e.g., reqres.in, petstore.swagger.io).*
*   Тема 2.4. Командная работа над кодом автотестов. Code Review в Pull Requests. Рефакторинг тестов. (4 ч)
    *   *Практикум: ревью кода автотестов одногруппника в GitHub.*
*   Тема 2.5. Введение в CI/CD для тестирования. GitHub Actions: автоматический запуск тестов при пуше в репозиторий. (4 ч)
    *   *Лабораторная работа: создание простого workflow .yml для запуска pytest.*

#### РАЗДЕЛ 3. ПРОДВИНУТЫЕ АСПЕКТЫ И ПРОЕКТНАЯ ДЕЯТЕЛЬНОСТЬ (60 часов)

ЧАСТЬ 3.1. УГЛУБЛЕННЫЕ ТЕМЫ (20 часов)
*   Тема 3.1.1. Регрессионное и интеграционное тестирование. Стратегии отбора тестов для регресса. (4 ч)
*   Тема 3.1.2. Тестирование баз данных. Проверка целостности данных через SQL-запросы. (4 ч)
*   Тема 3.1.3. Основы мобильного тестирования. Эмуляторы vs реальные устройства. Чек-лист особенностей. (4 ч)
*   Тема 3.1.4. Введение в нефункциональное тестирование (производительность, безопасность). Инструменты: Lighthouse, OWASP ZAP (демо). (4 ч)
*   Тема 3.1.5. Карьера в QA. Пути развития. Составление резюме и портфолио на GitHub. (4 ч)

ЧАСТЬ 3.2. КУРСОВОЙ ПРОЕКТ: ПОЛНЫЙ ЦИКЛ ТЕСТИРОВАНИЯ ВЕБ-ПРИЛОЖЕНИЯ (40 часов)
*   Этап 1 (8 ч): Выбор объекта тестирования (учебное приложение, например, "Менеджер задач"). Анализ требований. Разработка Тест-Плана. Заведение проекта в Qase/TestRail и Jira/GitHub Projects.
    *   *Результат:* Документ "Тест-План" в репозитории, созданный проект в инструментах.
*   Этап 2 (12 ч): Ручное тестирование. Создание полного набора тест-кейсов (50+). Проведение тестирования, заведение баг-репортов (минимум 10). Написание чек-листов для регресса.
    *   *Результат:* Набор тест-кейсов, список баг-репортов, отчет о ручном тестировании.
*   Этап 3 (12 ч): Автоматизация. Разработка набора автотестов (UI на Selenium и/или API) для покрытия ключевого функционала (15-20 тестов). Настройка CI через GitHub Actions.
    *   *Результат:* Код автотестов в репозитории, успешно работающий pipeline.
*   Этап 4 (8 ч): Итоговая отчетность и подготовка к защите. Сводка метрик. Подготовка Итогового отчета и презентации.
    *   *Результат:* Финальный отчет, презентация, полностью заполненный репозиторий на GitHub.
*   Защита проекта (4 ч): Публичная презентация результатов комиссии. Ответы на вопросы. Итоговая аттестация.
